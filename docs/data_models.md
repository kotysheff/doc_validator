# Модели данных DocValidator

Ниже описаны основные структуры данных, которые фактически используются в проекте.

## ExpectedFile

### Назначение
Описывает файл, который должен быть найден в целевом каталоге. Объект хранит имя файла, допустимые расширения и флаг обязательности.

### Поля
- `name` — базовое имя файла без расширения; длина от 1 до 255 символов; запрещены символы `\\ / : * ? " < > |`.
- `allowed_extensions` — список допустимых расширений; допустимы `.txt`, `.doc`, `.docx`, `.pdf`, `.xls`, `.xlsx`, `.ppt`, `.pptx`.
- `required` — признак обязательности файла (`True`/`False`).

### Пример
```python
ExpectedFile(name="report", allowed_extensions=[".pdf", ".docx"], required=True)
```

## ScannedFile

### Назначение
Описывает файл, который был найден во время сканирования каталога.

### Поля
- `name` — имя файла без расширения.
- `extension` — расширение файла с точкой.
- `size_bytes` — размер файла в байтах.
- `absolute_path` — объект `Path` с абсолютным путем.
- `relative_path` — объект `Path` с путем относительно каталога проверки.

## IssueType

### Назначение
Перечисление типов проблем, которые могут быть обнаружены во время проверки.

### Значения
- `MISSING_FILE` — обязательный файл отсутствует.
- `WRONG_EXTENSION` — файл найден, но его расширение не соответствует ожидаемому.
- `EMPTY_FILE` — файл найден, но он пустой.
- `EXTRA_FILE` — в каталоге обнаружен лишний файл.
- `INVALID_REQUIREMENTS` — структура или содержимое файла требований некорректны.
- `DIRECTORY_NOT_FOUND` — целевой каталог не существует.
- `REPORT_WRITE_ERROR` — ошибка при записи отчета.

## Severity

### Назначение
Уровни важности выявленных проблем.

### Значения
- `INFO`
- `WARNING`
- `ERROR`

## ReportFormat

### Назначение
Поддерживаемые форматы отчетов.

### Значения
- `TXT`
- `JSON`
- `CSV`

## ValidationStatus

### Назначение
Итоговый статус проверки.

### Значения
- `OK` — проблем не обнаружено.
- `WARNING` — есть предупреждения или информационные сообщения.
- `ERROR` — обнаружены критические ошибки.

## ValidationIssue

### Назначение
Описывает одну проблему, найденную во время проверки.

### Поля
- `issue_type` — тип проблемы из `IssueType`.
- `message` — текст сообщения.
- `severity` — уровень важности из `Severity`.
- `file_name` — имя файла, если проблема относится к конкретному файлу.

## ValidationResult

### Назначение
Содержит итог проверки: найденные, отсутствующие, ошибочные, пустые и лишние файлы, а также список проблем.

### Поля
- `status` — результат проверки (`ValidationStatus`).
- `found_files` — список найденных файлов, соответствующих требованиям.
- `missing_files` — список ожидаемых, но отсутствующих файлов.
- `wrong_extension_files` — файлы с неверным расширением.
- `empty_files` — пустые файлы.
- `extra_files` — лишние файлы.
- `issues` — список проблем.

## ValidationSettings

### Назначение
Настройки поведения проверки.

### Поля
- `recursive` — рекурсивный обход каталога.
- `check_empty_files` — проверять ли файлы на пустоту.
- `detect_extra_files` — выявлять ли лишние файлы.

## Requirements

### Назначение
Набор требований к проверяемому каталогу.

### Поля
- `required_files` — список объектов `ExpectedFile`.
- `settings` — объект `ValidationSettings`.
