# Модели данных

Ниже описаны основные структуры данных, используемые в проекте DocValidator.

## ExpectedFile

### Назначение
Описывает файл, который должен быть найден в целевом каталоге. Такие объекты обычно хранятся в списке и формируют набор требований к проверке.

### Поля
- `name` — базовое имя файла без расширения; допустимая длина — от 1 до 255 символов, а в названии запрещены символы `\ / : * ? " < > |`
- `allowed_extensions` — список допустимых расширений для файла; допустимые значения включают `.txt`, `.doc`, `.docx`, `.pdf`, `.xls`, `.xlsx`, `.ppt`, `.pptx`
- `required` — признак обязательности файла: `True`/`False`

### Пример
```python
name = "report"
allowed_extensions = [".pdf", ".docx"]
required = True
```

### Примечание

[//]: # (В дальнейшем планируется добавить возможность перечисления файлов для поиска через интерфейс командной строки. Для этого потребуется промежуточный слой преобразования аргументов в модель `ExpectedFile`.)

Правила формирования данных:
- `name` формируется на основе имени файла до точки
- расширения берутся после точки
- если для одного и того же `name` указано несколько расширений, они объединяются в список

## ScannedFile

### Назначение
Описывает файл, который был найден в целевом каталоге. Такие объекты хранятся в отдельном списке.

### Поля
- `name` — имя файла без расширения; инварианты совпадают с инвариантами модели `ExpectedFile`
- `extension` — расширение файла; инварианты совпадают с инвариантами модели `ExpectedFile`
- `absolute_path` — объект абсолютного пути к файлу (`Path`-объект)
- `size_bytes` — размер файла в байтах, целое неотрицательное число
- `relative_path` — путь к файлу относительно каталога поиска (`Path`-объект)

### Пример
```python
name = "report"
extension = ".txt"
path = Path(...) 
size_bytes = 400
relative_path = Path(...)
```

## IssueType

### Назначение
Перечисление типов проблем, которые могут быть обнаружены во время проверки.

### Значения
- `missing_file` — обязательный файл отсутствует
- `wrong_extension` — файл найден, но его расширение не соответствует ожидаемому
- `empty_file` — файл найден, но он пустой
- `extra_file` — в каталоге обнаружен файл, который не был указан в требованиях
- `invalid_requirements` — структура или содержимое файла требований некорректны
- `directory_not_found` — целевой каталог не существует
- `report_write_error` — ошибка при записи отчета

## Severity

### Назначение
Перечисление уровней важности проблем.

### Значения
- `info` — информационное сообщение
- `warning` — предупреждение, не блокирующее выполнение
- `error` — критическая ошибка, блокирующая корректное завершение проверки

## ReportFormat

### Назначение
Перечисление поддерживаемых форматов отчетов.

### Значения
- `txt`
- `json`
- `csv`

## ValidationStatus

### Назначение
Итоговый статус проверки в целом.

### Значения
- `OK` — проблем не обнаружено
- `WARNING` — есть предупреждения или информационные сообщения
- `ERROR` — обнаружена хотя бы одна критическая ошибка

## ValidationIssue

### Назначение
Описывает одну обнаруженную проблему в процессе проверки файла.

### Поля
- `issue_type` — тип проблемы; значение из перечисления `IssueType`
- `message` — подробное описание проблемы
- `file_name` — имя файла, к которому относится проблема; может быть пустым, если ошибка относится не к конкретному файлу
- `severity` — важность проблемы; значение из перечисления `Severity`

### Пример
```python
issue_type = IssueType.missing_file
message = "Необходимый файл не был найден по целевому пути"
file_name = "message.txt"
severity = Severity.error
```

## ValidationResult

### Назначение
Представляет итог результата проверки.

### Поля
- `status` — итоговый статус проверки; значение из перечисления `ValidationStatus`, вычисляется автоматически по списку `issues` после завершения проверки.
  - `OK`, если проблем нет
  - `WARNING`, если есть предупреждения или информационные сообщения
  - `ERROR`, если есть хотя бы одна критичная ошибка
- `found_files` — список объектов `ScannedFile`, представляющих файлы, найденные в каталоге и соответствующие запросу пользователя
- `missing_files` — список объектов `ExpectedFile`, описывающих файлы, которые должны были быть найдены, но отсутствуют
- `wrong_extension_files` — список объектов `ScannedFile`, представляющих файлы, для которых имя совпало с ожидаемым, но расширение оказалось неверным
- `empty_files` — список объектов `ScannedFile`, представляющих найденные файлы с нулевым размером
- `extra_files` — список объектов `ScannedFile`, представляющих файлы, найденные в каталоге, но не запрошенные пользователем;
- `issues` — список объектов `ValidationIssue`, описывающих проблемы, возникшие в ходе проверки

### Пример
```python
status = "OK"
found_files = [ScannedFile(name="primer1", extension=".docx", absolute_path=Path("..."), size_bytes=1200, relative_path=Path("primer1.docx"))]
missing_files = [ExpectedFile(name="missing", allowed_extensions=[".txt"], required=True)]
wrong_extension_files = [ScannedFile(name="wrong", extension=".py", absolute_path=Path("..."), size_bytes=100, relative_path=Path("wrong.py"))]
empty_files = [ScannedFile(name="empty_file", extension=".csv", absolute_path=Path("..."), size_bytes=0, relative_path=Path("empty_file.csv"))]
extra_files = [ScannedFile(name="extra", extension=".txt", absolute_path=Path("..."), size_bytes=150, relative_path=Path("extra.txt"))]
issues = [
    ValidationIssue(issue_type=IssueType.missing_file, message="Файл не был найден", file_name="missing.txt", severity=Severity.ERROR)
]
```
