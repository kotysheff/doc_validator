# Спецификация проекта DocValidator

## 1. Название проекта
DocValidator

## 2. Цель проекта
Создать инструмент для проверки каталога документов по заранее заданному набору требований и формирования отчета о найденных, отсутствующих, пустых и лишних файлах.

## 3. Целевая аудитория
Проект предназначен для системных администраторов, специалистов по документообороту и всех, кто работает с папками документов и хочет быстро проверить их соответствие заданным правилам.

## 4. Входные данные
Программа принимает:

- обязательный путь к каталогу, который нужно проверить;
- путь к JSON-файлу требований через параметр `-r` / `--requirements`;
- необязательные параметры CLI:
  - `-o`, `--output` — путь к файлу отчета;
  - `-f`, `--format` — формат отчета (`txt`, `json`, `csv`);
  - `--full` — расширенный вывод в консоль;
  - `--debug` — включение дополнительного логирования.

Также реализован GUI-интерфейс, который запускается через `python3 gui.py`.

## 5. Выходные данные
Программа формирует отчет и:

- либо выводит его в консоль;
- либо сохраняет в файл формата `.txt`, `.json` или `.csv`.

## 6. Что проверяется
Программа проверяет:

- наличие обязательных файлов;
- соответствие найденных файлов допустимым расширениям;
- наличие пустых файлов (при включенной настройке);
- наличие лишних файлов (при включенной настройке);
- корректность структуры JSON-файла требований;
- корректность пользовательского ввода и пути к файлам.

## 7. Ошибки и предупреждения
Программа обнаруживает следующие категории проблем:

- отсутствие обязательного файла;
- файл найден, но его расширение не соответствует ожидаемому;
- файл найден, но он пустой;
- в каталоге обнаружен лишний файл;
- ошибка загрузки требований;
- некорректный путь к каталогу или файлу;
- ошибка записи отчета.

## 8. Поддерживаемые форматы отчетов
В текущей версии поддерживаются:

- `.txt`
- `.json`
- `.csv`

Формат `.pdf` в реализации отсутствует.

## 9. Ограничения текущей версии
- проверка выполняется последовательно, без многопоточности;
- поддерживаются только расширения `.txt`, `.doc`, `.docx`, `.pdf`, `.xls`, `.xlsx`, `.ppt`, `.pptx`;
- отчеты и сообщения ориентированы на русский язык;
- интерфейс CLI и GUI реализован, но не предусматривает локализацию на другие языки.

## 10. Критерии готовности
Проект считается готовым, если:

- корректно обрабатываются валидные и невалидные входные данные;
- формируются отчеты в заявленных форматах;
- реализованы CLI и GUI-режимы;
- доступны автоматические тесты и инструкция по запуску.

## 11. План развития
- расширение списка поддерживаемых форматов файлов;
- улучшение UX GUI;
- добавление дополнительных проверок и отчетных метрик;
- дальнейшее развитие тестового покрытия.
