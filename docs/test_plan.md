# Тест-план проекта DocValidator

## Информация о документе
- Название: План тестирования проекта DocValidator
- Версия: 2.0
- Дата: 08.07.2026

## Цель тестирования
Проверить, что программа корректно:
- принимает требования из JSON-файла;
- сканирует каталог и выявляет отсутствующие, пустые, лишние и неверно расширенные файлы;
- формирует отчет в форматах `.txt`, `.json`, `.csv`, и `.pdf`;
- корректно работает как в CLI, так и в GUI.

## Объект тестирования
Приложение DocValidator в составе:
- CLI-модуля
- GUI-модуля
- модулей загрузки требований, сканирования, валидации и формирования отчетов

## Область тестирования
Включает:
- обработку аргументов командной строки;
- обработку файла требований;
- проверку существования целевого каталога;
- сканирование файлов в каталоге;
- проверку обязательных файлов и допустимых расширений;
- выявление пустых и лишних файлов;
- формирование отчетов в форматах `.txt`, `.json`, `.csv`, `.pdf`;
- запуск GUI и базовое взаимодействие с его элементами.

## Что не тестируется в текущем плане
- поддержка PDF-отчетов;
- многопоточная обработка;
- локализация на другие языки.

## Тестируемые сценарии

### 1. CLI
- запуск с обязательным каталогом и файлом требований;
- запуск с параметрами `--output`, `--format`, `--full`, `--debug`;
- запуск без файла требований;
- запуск с несуществующим каталогом;
- запуск с поврежденным JSON-файлом требований.

### 2. Загрузка требований
- чтение корректного JSON-файла;
- обработка отсутствующего JSON-файла;
- обработка JSON с отсутствующим полем `required_files`;
- обработка JSON с неверными значениями настроек.

### 3. Сканирование каталога
- поиск файлов в корне каталога;
- рекурсивный поиск по подкаталогам;
- обработка пустого каталога;
- обработка каталога с файлами без расширения.

### 4. Валидация
- выявление отсутствующих обязательных файлов;
- выявление неверных расширений;
- выявление пустых файлов;
- выявление лишних файлов при включенной настройке.

### 5. Формирование отчетов
- вывод в консоль;
- сохранение в `.txt`;
- сохранение в `.json`;
- сохранение в `.csv`;
- сохранение в `.pdf`

### 6. GUI
- запуск интерфейса через `python3 gui.py`;
- выбор каталога для проверки;
- ввод списка файлов;
- запуск проверки;
- сохранение отчета в файл через интерфейс.

## Виды тестирования
- функциональное;
- негативное;
- модульное (pytest);
- интеграционное;
- регрессионное.

## Тестовая среда
- Python 3.10+
- pytest
- tkinter (входит в стандартную библиотеку Python)
- Rich для форматированного вывода в консоль

## Критерии завершения тестирования
Тестирование считается завершенным, если:
- все основные сценарии выполняются без критических ошибок;
- отчеты формируются в заявленных форматах;
- CLI и GUI работают корректно в базовых сценариях;
- тесты проходят успешно.
