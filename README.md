# DocValidator

DocValidator — это CLI-программа для автоматизированной проверки каталога документов по заданному набору требований. Она помогает выявлять отсутствие файлов, несоответствие расширений, ошибки в структуре входных данных и формирует отчет о результатах проверки.

## Цель проекта

Создать удобный инструмент для проверки документов в папке по заранее заданным правилам с последующей генерацией отчета.

## Кто будет пользоваться

Проект предназначен для системных администраторов и специалистов по документообороту, работающих через командную строку.

## Основные возможности

- Проверка существования целевого каталога
- Проверка наличия файла требований или переданных файлов для поиска
- Сравнение фактических расширений файлов с ожидаемыми
- Обнаружение лишних файлов в каталоге
- Формирование отчета в формате `.txt`, `.json`, `.csv` или `.pdf`
- Поддержка режима отладки для вывода служебных сообщений

## Входные данные

Программа принимает:

- путь к каталогу для проверки
- путь к JSON-файлу требований или список файлов для проверки
- дополнительные ключи командной строки, такие как:
  - `--output <путь>` — сохранить отчет в файл
  - `--format <txt|json|csv|pdf>` — выбрать формат отчета
  - `--short` / `--full` — выбрать уровень детализации
  - `--debug` — включить отладочные сообщения

## Что программа проверяет

- наличие обязательных файлов
- корректность расширений файлов
- наличие лишних файлов
- структуру JSON-файла требований
- корректность пользовательского ввода

## Ограничения первой версии

- Поддерживаются файлы с расширениями `.docx`, `.doc`, `.pdf`, `.xlsx`, `.csv`, `.txt`, `.json`
- Отчеты сохраняются только в UTF-8
- Обработка выполняется последовательно, без многопоточности
- Поддерживается только русский язык интерфейса и отчетов
- GUI отсутствует, взаимодействие только через CLI
